페이지 개요

이 페이지는 김재현의 웹/블록체인 포트폴리오를 한곳에 정리한 자료입니다. 자체 운영 중인 PoA 네트워크 3개, Mint/Burn 브릿지, SBT·RWA 생성, Relayer 보안/가디언 복구 등 기술 스택과 산출물을 요약합니다.

용어 정리 — 토큰(ERC-20): 총량 한정 자산 / 코인: 네이티브 네트워크 수수료에 사용되는 화폐

Bridge Project (Mint/Burn)

Mint/Burn 방식의 체인 간 자산 이동. 현재 교환비는 1:1이나, 네트워크 성장 전략에 맞춰 1 MST = 100 JKK 등으로 조정 가능. 메타마스크 내 직접 사용은 제한되며, 제공 웹 페이지에서 교환 가능.

  • 연구실 6번 서버 PM2로 24시간 가동
  • 토큰·코인 모두 브릿지 가능하나, 코인 브릿지는 보안상 매우 취약 → 토큰 중심 모델 설계

브릿지 웹사이트

보안·운영 노트

  • 코인 브릿지: 보안상 매우 취약 → 토큰 브릿지 중심 비즈니스 모델
  • Relayer 24/7 구동(현재 6번 서버). 공개 리포지토리 사용 시 핵심 구간
  • 비밀정보(개인키·주소) 환경변수/로컬로 분리, 권한 최소화
  • 추가 보안 필요: 키 보관(HSM/HashiCorp Vault), 서명 분산(MPC), 감사 로그

제품화: SBT · RWA

SBT (SoulBound Token)

타인에게 전송되지 않는 비양도성 토큰. 신원·자격·소속 증명 등 활용. 관련 논문 보유, 구현 경험 기반으로 사업 계획서 전개 가능.

Non-Transferable Identity Governance

RWA (Real World Asset)

실물 자산 성격을 갖는 토큰. SBT와 개발 난이도는 유사하며, 권한·검증·거버넌스 설계가 핵심. 다양한 파트너십 모델로 사업화 가능.

Asset Token Compliance Use-case

NFT vs SBT/RWA

NFT는 “희소성 있는 토큰”에 대한 총칭이며, SBT·RWA는 기능/목적에 따른 NFT의 하위 유형으로 볼 수 있습니다.

그 외 핵심 기술

TPS·성능

  • 블록 타임 5초 / 현재 약 5 TPS 추정
  • TPS 증대는 구조적으로 고난이도 → 옵스 최적화·사용자 경험 먼저 개선

Relayer 보안

  • 24/7 가동, 공격 표면이 가장 큰 구간
  • 현재: 키/주소 환경변수 분리, 로컬 한정 실행
  • 향후: Vault/MPC, 접근제어, 서명감사·알림

가디언 복구

  • NFT/RWA/SBT 분실 시 가디언 승인으로 복구
  • 머스트리 3개 계정의 동의로 원 소유자 확인
  • 정책/UX 보완 개발 진행 필요

지갑·앱 구상

  • 웹 기반 지갑 앱으로 구매/교환/스테이킹 통합
  • 개발 공수 큼 — 점진적 MVP → 기능 확장 전략
